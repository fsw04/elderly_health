# 后端技术开发指导及规范（老人健康管理系统·MVP）

日期：2026-04-15

## 1. 后端职责边界
- 提供小程序与医生后台HTTP API
- 订阅MQTT消息：验签、防重放、幂等入库、错误落库
- 会话聚合：start/vitals/ecg/session_end
- 规则阈值报告生成：输出content_json
- 未体检提醒：3个月/6个月定时扫描、待办与触达记录
- 通知：站内通知必达；订阅消息可选（授权才发，失败不影响业务）
- 审计：医生端对用户档案/待办处理必须留痕

## 2. 推荐技术栈（可替换）
- Python Web：FastAPI（推荐）/ Django DRF / Flask
- ORM与迁移：SQLAlchemy+Alembic 或 Django ORM
- 数据库：PostgreSQL 或 MySQL 8
- 异步任务：Celery/RQ/Arq 或 DB表轮询worker（MVP可用）
- MQTT：paho-mqtt或等价库
- 缓存：Redis（用于token缓存、access_token缓存、任务队列）

## 3. 代码结构建议（按业务域）
- `auth/`：mp/admin鉴权、token、权限校验
- `users/`：用户、家属绑定、脱敏与加密
- `admin/`：医生端接口、审计
- `ingest/`：MQTT订阅、验签、幂等、错误表
- `exam/`：会话聚合、状态机、ECG完整性
- `report/`：规则阈值、content_json生成、规则版本管理
- `reminder/`：3/6个月扫描、待办处理
- `notify/`：站内通知、订阅消息发送与记录

## 4. 安全规范（MVP必做）
- 传输：HTTP/WS与MQTT均使用TLS
- 签名：MQTT使用HMAC-SHA256；包含topic与sentAt；sentAt做5分钟防重放窗口
- 幂等：`deviceId+msgId`唯一约束；重复直接返回成功不重复入库
- 隐私：身份证加密存储；返回脱敏；医生操作审计
- 权限：报告详情按report归属user二次校验（防reportId猜测越权）

## 5. 数据接入与会话状态机（约束）
- ECG策略：分片不完整则不生成报告
- 血糖编码：仅允许`GLU_FASTING`与`GLU_RANDOM`
- 会话状态：`in_progress` / `completed` / `partial` / `failed`
- 报告生成门禁：仅`completed`可生成；`partial/failed`禁止生成

## 6. 定时任务与去重（未体检提醒）
- last_exam_time：取最新completed会话的end_at
- 3/6个月阈值：按天（90/180）计算
- 去重：同一用户仅允许1条open待办；3M可升级为6M，不新建
- 冷却：done后7天内不重复创建

## 7. 订阅消息（可选增强）
- 用户授权状态落库，发送前先查
- 发送失败不影响站内通知与业务状态
- 开发环境建议支持mock发送

## 8. 观测与运维
- 健康检查：HTTP `/health` + MQTT订阅状态
- 关键指标：接入失败率、幂等命中率、报告生成耗时、待办生成数量
- 错误落库：`ingest_errors`保留raw_json与error_code，支持重放（可选）
