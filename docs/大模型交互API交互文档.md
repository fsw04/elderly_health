# 大模型交互API交互文档（预留）

日期：2026-04-15

## 1. 结论（MVP范围）
- 本项目MVP阶段不包含大模型能力
- 报告分析采用规则阈值引擎生成，不调用LLM
- 本文档作为后续迭代预留接口契约，避免未来引入LLM时破坏现有数据结构

## 2. 未来可选场景
- 报告解读：将结构化指标与异常项生成更口语化解释与生活方式建议
- 风险分层：基于历史趋势做“风险变化提示”（需数据积累与合规评估）
- 医生助手：医生端对报告的问答与总结（仅医生可见）

## 3. 交互原则（安全与合规）
- 不向模型发送身份证号、手机号等直接身份标识（仅发送脱敏/匿名字段）
- 模型输出必须带免责声明，不替代医生诊断
- 全量请求与响应需审计与可追溯（prompt版本、模型版本）

## 4. 预留API（示例）

### 4.1 生成报告解读
- `POST /api/ai/report-interpretation`
- 请求：
  ```json
  {
    "reportId": "rep_1",
    "contentJson": { "...": "..." },
    "audience": "elderly"
  }
  ```
- 返回：
  ```json
  {
    "code": "OK",
    "data": {
      "text": "本次体检显示血压偏高...",
      "highlights": ["建议复测", "减少高盐饮食"],
      "disclaimer": "本内容为健康管理参考，不替代医生诊断。"
    }
  }
  ```

### 4.2 生成医生端总结
- `POST /api/ai/doctor-summary`
- 请求：
  ```json
  { "reportId": "rep_1", "contentJson": { "...": "..." } }
  ```
- 返回：
  ```json
  { "code":"OK", "data": { "summary":"要点1...要点2..." } }
  ```
