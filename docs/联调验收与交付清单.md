# 联调验收与交付清单（MVP）

日期：2026-04-16

## 一、交付范围

- 后端：`FastAPI + SQLite`，覆盖医生端与小程序端核心接口。
- 小程序（老人/家属）：首页、报告、通知、个人资料、家属绑定全流程。
- 医生Web管理端：登录、用户管理、报告管理、待办处理、审计日志、报告打印。

## 二、启动命令

1. 启动后端

```bash
cd backend
.\venv\Scripts\python main.py
```

1. 启动医生端Web

```bash
cd doctor_admin_web
npm run dev

npm install
```

1. 启动小程序

- 使用微信开发者工具打开：`elderly_health_front/miniprogram`
- 本地后端地址默认：`http://localhost:8000/api`

## 三、联调顺序（建议）

1. 后端启动后访问 `http://localhost:8000/docs` 检查接口在线。
2. 医生端用默认账号登录并确认可拉取用户/报告列表。
3. 小程序首次登录，完成手机号绑定与个人资料更新。
4. 小程序发起家属申请，目标用户侧执行同意/拒绝。
5. 医生端处理待办，检查审计日志与通知回流。

## 五、验收用例（10条）

1. 医生登录成功

- 步骤：医生端输入默认账号密码登录。
- 预期：进入“未体检待办”页，无401报错。

1. 医生用户管理增改删

- 步骤：新增用户→编辑姓名→删除用户。
- 预期：列表数据同步变化，操作后有成功提示。

1. 医生报告详情

- 步骤：打开报告详情抽屉。
- 预期：显示摘要、异常项、规则命中、sections、suggestions。

1. 医生报告导出与打印

- 步骤：点击“导出JSON”“打印简版”。
- 预期：下载json成功；打印页有追踪号、页码、签名区。

1. 小程序首次登录与绑定

- 步骤：进入小程序，完成登录与绑定手机号。
- 预期：可正常进入首页，不再重复要求绑定。

1. 小程序报告浏览

- 步骤：进入报告列表→打开详情。
- 预期：列表与详情正常渲染，异常项可见。

1. 小程序通知已读

- 步骤：进入通知中心点击某条通知。
- 预期：通知状态更新为已读。

1. 家属绑定申请发起与取消

- 步骤：发起申请（pending）→点击取消申请。
- 预期：状态变更，不再占用待处理申请。

1. 家属绑定审批与解绑

- 步骤：目标用户同意申请→查看已绑定关系→执行解绑。
- 预期：关系建立成功后可解绑，解绑后列表移除。

1. 小程序分页与交互健壮性

- 步骤：在家属页反复加载更多、下拉刷新、连续点击操作按钮。
- 预期：有加载状态；防重复提交生效；失败提示为可读中文。
