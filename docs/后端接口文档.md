# 后端接口文档（老人健康管理系统·MVP）

日期：2026-04-17

## 1. 通用约定
- Base：`/api`
- 鉴权：`Authorization: Bearer <token>`
- 成功：`{ "code":"OK", "data": ... }`
- 失败：`{ "code":"XXX", "message":"..." }`
- 分页：`{ page, pageSize, total, items }`

## 2. 小程序API（mp）

### 2.1 登录与注册
- `POST /api/mp/login`
  - 请求：`{ "code": "<wx.login code>" }`
  - 返回：`{ token, expiresAt, needBindPhone }`
- `POST /api/mp/account/login`
  - 请求：`{ "phone": "13800000000" }`
  - 返回：`{ token, expiresAt, needBindPhone:false }`
- `POST /api/mp/account/register`
  - 请求：`{ "phone": "13800000000", "name": "张三", "gender": "M|F|U", "birthDate": "YYYYMMDD", "currentAddress"?: "现住址" }`
  - 返回：`{ token, expiresAt, needBindPhone:false }`
- `POST /api/mp/sms/send`
  - 请求：`{ "phone": "13800000000", "scene": "BIND_PHONE" }`
- `POST /api/mp/bind-phone`
  - 请求：`{ "phone": "13800000000", "smsCode": "123456" }`

### 2.2 个人资料
- `GET /api/mp/me`
  - 返回补充：`age`（后端按`birthDate`自动计算）、`currentAddress`
- `PUT /api/mp/me`
  - 请求：`{ name?, gender?, birthDate?, idCard?, currentAddress? }`

### 2.3 首页聚合
- `GET /api/mp/home`
  - 返回：`me + counters + recentReport`

### 2.4 报告
- `GET /api/mp/reports?page=&pageSize=&onlyAbnormal=0|1&targetUserId=可选`
  - 说明：不传targetUserId为本人；传targetUserId为家属视角（需授权）
- `GET /api/mp/reports/{reportId}`
  - 说明：必须按report归属user做二次授权校验（本人或家属已绑定）

### 2.5 通知
- `GET /api/mp/notifications?page=&pageSize=&onlyUnread=0|1`
- `POST /api/mp/notifications/{id}/read`
- `POST /api/mp/notifications/read-all`（可选）

### 2.6 订阅消息授权
- `POST /api/mp/subscribe-settings`
  - 请求示例：
    ```json
    {
      "settings": [
        { "templateKey": "REPORT_READY", "status": "accept" },
        { "templateKey": "EXAM_REMIND", "status": "accept" }
      ]
    }
    ```

### 2.7 家属绑定（两阶段）
- `POST /api/mp/family-link-requests`
  - 请求：`{ "targetPhone": "13800000000", "relationType"?: "child" }`
- `GET /api/mp/family-link-requests?role=target|requester&status=pending|rejected|expired&page=&pageSize=`
- `POST /api/mp/family-link-requests/{requestId}/approve`
- `POST /api/mp/family-link-requests/{requestId}/reject`
  - 请求：`{ "note"?: "..." }`
- `POST /api/mp/family-link-requests/{requestId}/cancel`
- `GET /api/mp/family-links?page=&pageSize=`
- `DELETE /api/mp/family-links/{linkId}`

## 3. 医生端API（admin）

### 3.1 登录
- `POST /api/admin/login`

### 3.2 用户管理
- `GET /api/admin/users?q=&page=&pageSize=`
  - 说明：`q`支持姓名/手机号/证件号/现住址模糊搜索
- `POST /api/admin/users`
  - 请求补充：`currentAddress?`
- `GET /api/admin/users/{id}`
  - 返回补充：`age`、`currentAddress`
- `PUT /api/admin/users/{id}`
  - 请求补充：`currentAddress?`
- `DELETE /api/admin/users/{id}`

### 3.3 会话与报告
- `GET /api/admin/users/{id}/sessions?page=&pageSize=`
- `GET /api/admin/users/{id}/reports?page=&pageSize=`
- `GET /api/admin/reports?from=&to=&riskLevel=&q=&page=&pageSize=`
- `GET /api/admin/reports/{reportId}`
- `PUT /api/admin/reports/{reportId}/doctor-summary`
  - 说明：医生端编辑“基础体检信息”，未收集字段可传空字符串
  - 请求：
    ```json
    {
      "height": "153.5 cm",
      "weight": "43.6 kg",
      "bmi": "18.5 kg/m²",
      "bloodPressure": "115/71 mmHg",
      "fastingBloodGlucose": "5.23 mmol/L",
      "ecgFinding": "窦性心动过缓；T波低平",
      "bUltrasound": "",
      "tcmConstitution": "平和质:是"
    }
    ```
  - 返回：更新后的`ReportResponse`

### 3.4 未体检待办
- `GET /api/admin/reminder-tasks?status=open|done&type=NO_EXAM_3M|NO_EXAM_6M&q=&page=&pageSize=`
- `POST /api/admin/reminder-tasks/{taskId}/handle`
  - 请求：`{ "action": "notify|call|other", "channel": "inapp|wechat_subscribe|phone|other", "note": "..." }`

### 3.5 审计
- `GET /api/admin/audit?targetType=user&targetId=&page=&pageSize=`

## 4. 错误码（MVP核心）
- `MP_412_PHONE_REQUIRED`：未绑定手机号
- `AUTH_403_FORBIDDEN`：无权限
- `SMS_429_RATE_LIMIT`：验证码频控
- `SMS_400_CODE_INVALID` / `SMS_410_CODE_EXPIRED`
- `FAM_409_PENDING_EXISTS` / `FAM_410_REQUEST_EXPIRED`
- `ING_401_SIG`：MQTT签名失败
- `ING_408_TS_EXPIRED`：sentAt超窗（防重放）
- `ING_200_DUP`：幂等重复
- `ING_409_CHUNK`：ECG分片缺失/校验失败

## 5. MQTT数据接入协议（摘要）

### 5.1 Topic
- 上行：`fc/{fcId}/up`、`watch/{watchId}/up`、`sensor/{sensorId}/up`

### 5.2 通用payload字段
- `deviceId`, `deviceType`, `msgId`, `ts`, `sentAt`, `type`, `sessionId?`, `ver`, `data`, `sigAlg`, `sigVer`, `sig`

### 5.3 type枚举（MVP）
- `device_meta`, `session_start`, `vitals`, `ecg_meta`, `ecg_chunk`, `session_end`

### 5.4 血糖编码（基线）
- 仅允许：`GLU_FASTING` 与 `GLU_RANDOM`

### 5.5 ECG规则（基线）
- 若会话出现ECG（meta或标记ecgPresent）：必须收齐全部chunk且校验通过
- 否则会话=partial，禁止生成reports与“报告已生成”通知

## 6. MQTT签名规范（HMAC v1）
- `sigAlg=HMAC-SHA256`，`sigVer=v1`
- 防重放：使用`sentAt`做5分钟窗口；`ts`为业务采集时间可很早（离线补传）
- canonical string（`\n`换行，固定顺序）：
  - `deviceId=<deviceId>`
  - `ts=<ts>`
  - `sentAt=<sentAt>`
  - `msgId=<msgId>`
  - `type=<type>`
  - `sessionId=<sessionId或空>`
  - `ver=<ver或1.0>`
  - `topic=<实际MQTT topic>`
  - `data=<canonical_json(data)>`
- `sig = base64(HMAC_SHA256(secret, canonical_string_utf8))`
- `canonical_json`：JSON对象key排序、无空格、数组顺序不变

## 7. 报告content_json规范（schemaVersion=1.0）
- 前端只渲染，不做业务计算
- 顶层字段：`schemaVersion, reportId, sessionId, generatedAt, user, meta, summary, abnormalities, sections, suggestions, ruleVersion`
- 新增医生补录字段：`doctorSummary`（对象，允许空字符串）
  - `height, weight, bmi, bloodPressure, fastingBloodGlucose, ecgFinding, bUltrasound, tcmConstitution`
- `summary.riskLevel`：`low/medium/high`
- `sections`顺序建议：vitals → glucose → weight → ecg（空模块隐藏）
- `abnormalities`：后端输出并排序（danger > high > low）

## 8. 验收用例（摘要）
- 体检闭环：start→vitals→end → 生成报告 → 小程序可查看
- 幂等：重复`deviceId+msgId`不重复入库、不重复生成报告
- 血糖：`GLU_FASTING`与`GLU_RANDOM`分别按对应阈值判级
- ECG缺片：不生成报告、不推“报告已生成”通知
- 家属：申请→老人确认→家属只读查看报告；解绑后立即403
- 未体检：3/6个月生成待办→医生处理→用户通知中心可见提醒记录
