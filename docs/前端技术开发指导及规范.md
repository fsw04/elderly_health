# 前端技术开发指导及规范（老人健康管理系统·MVP）

日期：2026-04-15

## 1. 端侧范围
- 微信小程序：老人/家属用户端
- Web管理端：医生后台（PC浏览器）

## 2. 技术选型（建议）

### 2.1 小程序
- 原生小程序（WXML/WXSS/JS）或基于团队熟悉的框架（不强制）
- 重点：接口封装、登录态守卫、错误态统一处理、分页列表组件复用

### 2.2 医生Web后台
- 方案A（推荐）：Vue3 + Vite + TypeScript + Element Plus
- 方案B：React + Vite + TypeScript + Ant Design
- 目标：表格/表单/分页/筛选/弹窗交互成熟，减少自研组件

## 3. 目录与模块建议
- `api/`：接口请求封装（统一baseURL、token注入、错误码映射）
- `pages/`：页面（按业务域划分：report、notification、profile、family、users、reminders）
- `components/`：复用组件（列表空态、错误态、风险徽标、指标行、折叠面板）
- `utils/`：格式化（时间、脱敏、数值保留位）

## 4. 登录态与路由守卫（小程序）
- 启动：`wx.login`换取token，写入本地缓存
- 进入受保护页面前统一检查：
  - 未登录：重登
  - 未绑定手机号：跳转绑定页
- 401统一处理：清token → 触发登录流程

## 5. 页面规范（小程序MVP）

### 5.1 页面清单
- 首页：最近一次体检摘要 + 未读通知数 + 待处理家属请求
- 报告列表：轻量摘要列表 + 仅异常筛选（可选）
- 报告详情：按content_json渲染（summary → abnormalities → sections → footer）
- 通知中心：全部/未读tab + 订阅消息授权入口
- 绑定手机号：短信验证码 + 频控
- 资料完善：姓名/性别/生日（身份证可选，展示脱敏）
- 家属绑定：发起申请、查看状态、老人确认页、已绑定列表与解绑

### 5.2 报告渲染契约（前端原则）
- 前端不计算风险等级与异常项，只渲染后端返回字段
- 异常置顶只使用`abnormalities`数组
- sections顺序固定：vitals → glucose → weight → ecg（空模块隐藏）

### 5.3 数值格式化规范
- 统一格式化函数：根据code决定保留小数位与单位
- 缺失值统一展示`--`

## 6. 医生Web页面规范（MVP）
- 路由：/users /reports /reminders /audit
- 表格：分页、关键筛选项、空态、加载态、错误态统一
- 表单：新增/编辑用户弹窗；处理待办弹窗（动作、渠道、备注）
- 报告详情：展示content_json + ruleHits（医生视角专属）

## 7. 接口调用规范
- 所有接口调用必须走统一封装（注入token、超时、重试策略）
- 错误码映射到用户可读文案：
  - `MP_412_PHONE_REQUIRED`：请先绑定手机号
  - `AUTH_403_FORBIDDEN`：暂无权限查看
  - `SMS_429_RATE_LIMIT`：操作频繁，请稍后再试
- 列表接口统一分页结构

## 8. 安全与隐私（前端侧）
- 身份证/手机号统一脱敏展示，不在前端日志打印敏感信息
- 分享能力MVP不做或仅分享“入口”，不分享具体报告内容链接
